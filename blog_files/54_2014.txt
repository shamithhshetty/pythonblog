<h3 class="post-title entry-title">
<a href="https://pythoninsider.blogspot.com/2014/03/python-340-released.html">Python 3.4.0 released</a>
</h3>
<div class="post-header">
<div class="post-header-line-1"></div>
</div>
<div class="post-body entry-content">
<a href="http://www.python.org/download/releases/3.4.0/">Python 3.4.0</a> has been released today.<br/>
<br/>
Python 3.4 includes a range of improvements of the 3.x series, including hundreds of small improvements and bug fixes.  Major new features and changes in the 3.4 release series include:<br/>
<ul><li>PEP 428, a "pathlib" module providing object-oriented filesystem paths<br/>
<li> PEP 435, a standardized "enum" module<br/>
<li> PEP 436, a build enhancement that will help generate introspection information for builtins<br/>
<li> PEP 442, improved semantics for object finalization<br/>
<li> PEP 443, adding single-dispatch generic functions to the standard library<br/>
<li> PEP 445, a new C API for implementing custom memory allocators<br/>
<li> PEP 446, changing file descriptors to not be inherited by default in subprocesses<br/>
<li> PEP 450, a new "statistics" module<br/>
<li> PEP 451, standardizing module metadata for Python's module import system<br/>
<li> PEP 453, a bundled installer for the *pip* package manager<br/>
<li> PEP 454, a new "tracemalloc" module for tracing Python memory allocations<br/>
<li> PEP 456, a new hash algorithm for Python strings and binary data<br/>
<li> PEP 3154, a new and improved protocol for pickled objects<br/>
<li> PEP 3156, a new "asyncio" module, a new framework for asynchronous I/O<br/>
</li></li></li></li></li></li></li></li></li></li></li></li></li></li></ul>
<p>The <a href="http://docs.python.org/3.4/whatsnew/3.4.html">full "What's New" document</a> lists all major changes.
<div style="clear: both;"></div>
</p></div>
